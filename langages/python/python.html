Conditions

Une condition renvoie True si elle est respectée et False sinon

Opérateurs de comparaison

Condition Description de ce que vérifie la condition
a == b a égal à b
a < b a strictement inférieur à b
a > b a strictement supérieur à b
a <= b a supérieur ou égal à b
a != b a n’est pas égal à b
a in b a est présent dans b (qui peut être une liste)
a is None Tester si une variable est nulle
or À mettre entre deux conditions, permet d’avoir une des deux conditions qui doit être vraie
and À mettre entre deux conditions, permet d’avoir deux conditions qui doivent être vraie
not condition Ne doit pas respecter la condition

Tests de conditions

Instruction Description
if condition1 :
    instruction1		 Si condition1 est vraie, alors on exécute instruction1
if condition1 :
    instruction1
else :
    instruction2		 Si condition1 est vraie, alors on exécute instruction1, sinon, on exécute instruction2
if condition1 :
    instruction1
elif condition2 :
    instruction2
else :
    instruction3		 Si condition1 est vraie, alors on exécute instruction1, sinon, si condition2 est vraie, on exécute instruction2, sinon, on exécute instruction3
try :
    instruction1
except :
    instruction2		 Si instruction1 provoque une erreur, on exécute instruction2

Boucles

Instruction Description
for i in range(n) :
    instruction1		 On répète n fois l’instruction pour i allant de 0 compris à n non compris
for i in range(d, f) :
    instruction1		 On répète f-d fois l’instruction pour i allant de d compris à f non compris
for i in range(d, f, p) :
    instruction1		 On répète (f-d)/p fois l’instruction pour i allant de d compris à f non compris avec pour pas égal à p
for elt in liste :
    instruction1		 On parcourt la liste (ou une chaîne de caractères) pour elt prenant toutes les valeurs de la liste
while condition :
    instruction1		 On répète jusqu’à ce que la condition soit fausse (peut ne pas être répété)
break Permet de sortir d’une boucle sans la terminer (à éviter si possible)
pass Passer un bloc s'il est vide, pour éviter des erreurs (à éviter si possible)

Les fonctions

Les fonctions peuvent être situées dans le même fichier, mais doivent être définies avant d’être appelées.

Créer une fonction

def maFonction(variable1, variable2…) :
    instructions

Retourner une plusieurs valeurs de la fonction (non obligatoire)

return variable

ou

return variable1, variable2…

Faire un appel à la fonction

variable = maFonction(valeur1, valeur2…)

ou

variable1, variable2… = maFonction(valeur1, valeur2…)

ou (s'il n'y a pas de variable de retour)

maFonction(valeur1, valeur2…)

Remarque : Il est possible d'affecter une valeur par défaut, si aucune valeur n'est fournie aux paramètres de la fonction, sous la forme : variable = valeurParDefaut

Les classes

Les classes peuvent être situées dans le même fichier, mais doivent être définies avant d’être appelées.

Les classes de base

Créer une classe

class MaClasse :

    variableStatique3 = valeur3

    def __init__(self, variable1, variable2…) :
        self.variable1 = variable1
        self.variable2 = variable2

Définir la classe dans un objet

monObjet = MaClasse(valeur1, valeur2…)

Ajouter un retour de la classe lors de la conversion en chaîne de caractères

def __str__(self) :
    return "message"

Créer une méthode (dans une classe)

def maMethode(self, variable1, variable2…) :
    instructions

Faire un appel d’une méthode autre que __init__ et __str__ ou d’une variable dans une classe

self.maMethode(valeur1, valeur2…)

self.variable1 = valeur1

Faire un appel d’une méthode autre que __init__ et __str__ ou d’une variable en dehors d'une classe

monObjet.maMethode(valeur1, valeur2…)

monObjet.variable1 = valeur1

Faire un appel d’une variable statique dans une classe et en dehors

MaClasse.variableStatique3 = valeur3
